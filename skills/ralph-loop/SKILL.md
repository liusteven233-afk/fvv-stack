# Ralph Loop

可自动循环执行任务，持续搜索、迭代优化，直到任务完成。

## 功能

- **自动循环**: 设定任务后自动反复执行直到完成
- **迭代优化**: 每次循环基于上次结果优化
- **持续搜索**: 在循环中持续搜索更好的方案
- **完成判定**: 自动判断任务是否完成，避免无限循环

## 使用方式

### 基本用法
在cronjob或delegate_task中使用循环逻辑：

```
任务：优化函数性能
循环：每次运行profiler，找到瓶颈，优化，重新测试
终止条件：性能提升小于5% 或 超过10轮
```

### 示例：优化代码
```
循环任务: 分析 ~/app.py 的性能瓶颈
每次循环:
  1. 运行 cProfile 获取性能数据
  2. 找出最耗时的3个函数
  3. 优化其中一个
  4. 重新运行测试
  5. 如果性能提升 < 3% 或循环超过8次 → 停止
```

### 示例：搜索最佳参数
```
循环任务: 找到最优的价格策略
每次循环:
  1. 计算6国到账
  2. 找出到账最高的国家
  3. 调整参数（佣金、运费）
  4. 如果改进 < 1% → 停止
```

## 注意事项

⚠️ **费用控制**: 长时间循环可能消耗大量API配额。建议：
- 设最大循环次数（建议≤10）
- 每次循环前检查剩余token/配额
- 使用`notify_on_complete`获取结束通知
- 不要用于无限等待的任务

## 适用场景

| 场景 | 说明 |
|------|------|
| 代码优化 | 反复分析-修改-测试 |
| 参数调优 | 网格搜索/迭代逼近最优值 |
| 数据清洗 | 重复清洗直到数据达标 |
| 多轮研究 | 基于上次结果深入搜索 |
| A/B测试 | 自动运行对比试验 |

# FVV Stack v2 — 美客多全流程运营系统 (修正版)

## 核心工作流
```
[ML前端选品] → 1688找货源 → 核价 → 上架ML → 优化listing → 发货 → 售后
   │                │          │        │        │           │       │
   │                │          │        │        │           │       │
  ML搜索页       1688手动    自动计算  API上架   AI生成     采购记录  多账号
  7天爆款        搜索SKU     6国利润  批量发   标题/描述   物流追踪  统一管理
  30天新品                                             主图扒取

```

## 模块详细设计

### 1️⃣ 选品 — ML前端数据提取 (仿智赢蓝鲸)

**核心逻辑**: 不是爬1688，而是**爬美客多前端页面**，提取热门新品数据

**数据来源**:
- ML搜索页 (listado.mercadolibre.com.mx/...)
- ML类目页
- ML搜索结果排序（按新品、按销量）

**选品策略**:
| 筛选条件 | 说明 |
|---------|------|
| 📅 上架30天内 | 新品才有红利期 |
| 🔥 7天销量高 | 市场验证过的爆款 |
| 💰 价格区间 | 排除超低价/超高价 |
| 📦 跨境可发 | 非禁运品类（电子/液体/大件过滤） |
| 👥 竞争度低 | TOP10卖家集中度低 |

**数据字段**:
```json
{
  "ml_item_id": "MLM123456",
  "title": "Cargador USB-C 65W",
  "price_ml": 349,
  "currency": "MXN",
  "category": "Cargadores",
  "category_id": "MLM1234",
  "listed_date": "2026-04-15",
  "sales_7d": 850,
  "total_sales": 1200,
  "seller_name": "TechShopMX",
  "seller_type": "mercadolibre",
  "images": ["https://http2.mlstatic.com/..."],
  "shipping_free": true,
  "competitors": 120,
  "top10_market_share": 65
}
```

**实现方式**: 
- Chrome扩展内容脚本（在用户已登录的ML页面运行）
- 或者后端Playwright爬虫
- 评分算法：`score = sales_7d × weight1 + (30-days_since_list)/30 × weight2 - competitor_count × weight3`

**用户操作流**:
```
打开ML → 搜索关键词 → 扩展自动提取结果 → 按条件过滤
→ 显示"选品池" → 点中商品 → 自动在1688搜索同款
→ 显示1688匹配结果 → 确认货源 → 进入核价
```

### 2️⃣ 1688找货源 — 关联匹配

不是爬所有1688，而是**用户选了ML爆款后，系统帮在1688找同款**

**流程**:
1. ML选中商品 → 提取标题关键词
2. 在1688搜索同款 → 返回匹配结果
3. 显示价格对比：ML售价 vs 1688货源价
4. 用户确认货源 → 保存关联关系

**注意**: 1688数据靠扩展内容脚本（用户在浏览器已登录1688），不走后端爬虫

### 3️⃣ 核价 — 复用现有代码

- 现有 fvv dashboard 的核价引擎（5种利润模式）
- 6国运费表 + 阈值高低双价
- 汇率实时/手动
- 输出：各国售价、到账、利润率

### 4️⃣ 上架 — ML API自动上架

**整体架构模仿现有ML自动上架工具**:

```
[扩展侧面板] → 填写商品信息
  → 类目选择（ML API GET /categories）
  → 属性填写（尺寸/颜色/品牌等）
  → 图片上传（ML图床）
  → 价格输入（可自动填充核价结果）
  → 点击"上架" → ML API POST /items
  → 返回listing链接
```

**多账号**: OAuth token池，每个账号独立token
**批量上架**: CSV导入 → 队列逐一提交

### 5️⃣ 优化Listing

| 优化项 | 具体做法 |
|--------|---------|
| 🖼️ **主图扒取** | 扩展在ML详情页提取竞品主图原图URL→下载到本地 |
| 🖼️ **主图优化** | AI去除背景/加水印/调整尺寸/Mercado Libre标准 |
| 📝 **描述优化** | AI生成西语/葡语描述，**不含品牌名/其他店铺名/保修信息** |
| 🏷️ **标题优化** | 提取ML热搜词 → AI组合标题 → 关键词密度检测 |

**描述规则（重点）**:
- ❌ 不能出现品牌名（除非是自己注册的品牌）
- ❌ 不能出现其他店铺信息
- ❌ 不能出现保修承诺（除非在ML系统内设置）
- ✅ 只写：产品功能、规格参数、适用场景、发货说明
- ✅ 西语/葡语本地化（非机翻）

### 6️⃣ 发货 — 采购记录+物流追踪

**我不需要一个外部系统来分析** — 你是让我做**一个录入系统**：

```
用户填写 → 订单号、物流单号、1688采购单号
系统 → 自动关联订单、追踪物流状态、计算时效
```

**功能**:
- 📋 采购记录表：ML订单 ↔ 1688采购单 ↔ 物流单号
- 🔗 自动关联：输入ML订单号 → 拉取ML订单信息
- 🚚 物流追踪：快递100/4PX API 自动更新状态
- ⏱️ 时效分析：从采购到签收各环节耗时统计
- 📊 异常预警：超时未发货/物流异常自动提醒

### 7️⃣ 售后 — 多账号统一管理

**目标**: 登录多个ML账号 → 一个界面管理所有售后

**实现**:
- OAuth token池管理多个ML卖家账号
- 定时拉取各账号的：
  - 未读消息（questions）
  - 退换货请求
  - 纠纷/投诉
- 统一消息面板，支持：
  - 西语/葡语快捷回复模板
  - 批量处理
  - 回复率/响应时间统计
